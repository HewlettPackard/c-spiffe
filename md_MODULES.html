Modules

The project structure is organized into modules:

  • **bundle** facilities for manipulating bundles.
  • **federation** facilities for retrieving and serving federated bundles.
  • **internal** general use utility functions.
  • **logger** code for logging functions.
  • **protos** ProtoBuf definition for the SPIFFE Workload API. C++ GRPC code is generated from this.
  • **spiffeid** facilities for manipulating spiffe IDs and Trust Domains.
  • **spiffetls** Functions for using TLS with SPIFFE.
  • **svid** utility functions for SVIDs.
  • **utils** general use utility functions.
  • **workload** facilites for fetching SVIDs and Bundles from the Workload API.
